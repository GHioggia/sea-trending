# HTML 报告设计文档

## 设计定位

**情报简报风格** — 面向游戏策划团队的每日趋势速览。
信息密度高，层次分明，适合每天打开花 2-3 分钟快速扫读。

## 模块说明

### 1. Header（sticky 顶栏）
- Logo + 标题 + 日期
- 右侧统计：总条数、源正常数、跨国热点数、洞察数
- sticky 定位，滚动时常驻可见

### 2. 数据源健康条
- 每个源一个绿/红圆点
- 右侧显示去重统计和分类计数
- 一目了然判断今日数据质量

### 3. 三国速览（首屏核心）
- 三列 grid，每国一个 card
- 每 card 显示 top 4 条目，带分类 badge
- 非中文条目带括号中文注释
- 扫一眼就知道三个国家今天发生了什么

### 4. 跨国共同热点
- 出现在 ≥2 个国家的话题
- 卡片展示：关键词、国家 tags、综合评分、四维评分 pills
- 跨国热点是最有战略价值的信息

### 5. 国家详情面板
- 每国一个面板，顶部颜色条区分（PH蓝/ID红/TH黄）
- 面板内按四分类展示
- 每条 item：排名 → 关键词(中文) → 摘要 → 来源tag → 评分pills
- 只展示有分值的评分（避免显示 0.00）

### 6. 游戏设计洞察
- 卡片网格布局
- 每张卡片：关键词 + 国家tag + 四个维度（关注原因/玩家心理/设计方向/风险）
- 左边框颜色标识来源分类
- 洞察必须引用具体 item，不编造

### 7. 风险提醒
- 红色底色区域，醒目警示
- 按风险类型分组：自然灾害/政治/民生/宗教
- 每项有风险条形图直观展示风险等级

### 8. 数据源状态表
- 表格展示所有源的运行状态
- 列：源名 / 平台 / 方式 / 状态 / 各国条数

### 9. 运行日志
- 等宽字体展示关键运行参数
- 日期、耗时、国家、去重、错误

## 排版原则

- **字体**: 系统字体栈 + Noto Sans SC 兜底
- **等宽**: SF Mono / Fira Code（数字、评分、日志）
- **行高**: 1.6（正文）
- **最大宽度**: 1200px 居中
- **响应式断点**: 768px（三列 → 单列）

## 设计决策

1. **深色主题** — 策划日常工作大量看屏幕，深色减少眼疲劳；信息类仪表盘深色更专业
2. **评分 pills 而非图表** — 紧凑，不占空间，数据量不大时比雷达图更快阅读
3. **非中文括号注释** — 中文团队快速理解外语关键词，不影响原始关键词的可搜索性
4. **风险区域红底** — 强烈视觉提示，避免策划忽略敏感话题
5. **不用 JS 交互** — 单 HTML 文件离线可读，无折叠展开

## 模板化建议（下一步 6B）

### Jinja2 模板化策略

1. 将 `public/design-preview.html` 拆分为 `templates/daily_report.html`
2. 所有 mock 数据替换为 Jinja2 变量：
   ```jinja
   {{ report.date }}
   {{ report.total_items }}
   {% for summary in report.country_summaries %}
   {% for section in report.sections %}
   {% for insight in report.trend_summary.design_insights %}
   ```
3. 评分 pills 用 macro：
   ```jinja
   {% macro score_pill(label, value, cls) %}
   {% if value > 0 %}
   <span class="score-pill {{ cls }}">{{ label }} {{ "%.2f"|format(value) }}</span>
   {% endif %}
   {% endmacro %}
   ```
4. 中文注释用 translator 在 Python 端预处理，模板只接收 `item.keyword_zh`
5. CSS 保持内联（单文件可离线查看）

### 需要扩展 report.json 的字段

不需要。当前 schema 已包含所有需要的数据：
- `sections[].items[].scores` → 评分 pills
- `country_summaries` → 三国速览
- `trend_summary.cross_country_hotspots` → 跨国热点
- `trend_summary.design_insights` → 设计洞察
- `trend_summary.gaming_hotspots` → 游戏热点

### 风险提醒模块数据

当前 report.json 没有独立的风险数据结构。建议在 `analyzer.py` 中新增 `extract_risk_items()` 函数：
- 筛选 risk_score > 0 的 items
- 按风险类型分组（灾害/政治/民生/宗教）
- 输出到 `report.trend_summary.risk_alerts`
